## What Field Scout does

Field Scout is a **free** Atlassian Forge app for Jira Cloud that gives administrators a centralized dashboard to analyze and manage custom-field usage across their entire instance.

Jira Cloud enforces a **700 custom field limit**. Over time, fields accumulate — from plugins, migrations, and changing workflows — and many end up unused, duplicated, or forgotten. Field Scout scans every custom field, classifies it by usage, and shows you exactly where you stand and which fields are safe to clean up.

## The problems Field Scout solves

**"We're approaching the 700 custom field limit."**  
Instantly see your total field count with visual warnings as you approach or exceed the limit.

**"I don't know which fields are actually being used."**  
Every field is classified as Active, Stale, Likely Unused, or Unused based on real usage data from Jira.

**"Old fields from removed plugins are cluttering our instance."**  
Filter by origin to see fields from ScriptRunner, Tempo, JSM, and other plugins — even after the plugin is removed.

**"I need to audit fields before a migration."**  
Export your entire field inventory to CSV with all metadata — screens, contexts, last-used dates, classifications.

**"Some fields have no description and nobody knows what they do."**  
Toggle the *Missing Description* filter to instantly find undocumented fields that need attention.

**"I need to report on field hygiene to management."**  
The Field Health Score gives you a single percentage that summarizes your instance's field hygiene, with actionable cleanup guidelines.

**"We have duplicate fields with similar names."**  
Potential Duplicates detection finds exact-match and similar-name fields across your instance, including team-managed projects.

## Key features

- **Field Health Score** — Single percentage that summarizes your instance's field hygiene at a glance.
- **Field classification** — Every custom field is automatically classified as Active, Stale (>6mo, >1y), Likely Unused, or Unused based on real usage data.
- **Actionable guidelines** — Prioritized recommendations with clear ACTION, REVIEW, and TIP labels.
- **Summary dashboard** — Cards showing health score, total field count, system fields, team-managed fields, unused fields, likely unused, and stale fields.
- **Potential duplicates** — Spot fields with identical or similar names that could be consolidated, with filtering by match type and field type.
- **Fields by Origin** — Understand which products and plugins created your fields, with per-origin health breakdowns in table or chart views.
- **Fields by Type** — See how your fields are distributed across data types (text, select, date, etc.) in table, chart, or tag-cloud views.
- **Fields per Project** — See how many fields each project uses and which are unused.
- **Field Scope** — Know whether each field applies to all projects, only team-managed, or only company-managed.
- **Origin tracking** — See which product or plugin created each field: Jira Core, Jira Software, JSM, ScriptRunner, Tempo, Advanced Roadmaps, and more.
- **Comprehensive filtering** — Name, status, field type, origin, scope, last-used recency. Toggles for locked fields, missing descriptions, zero screens, zero contexts.
- **Direct admin links** — Click any field name to jump to its configuration page in Jira admin.
- **CSV export** — Filtered view or full inventory, ready for spreadsheets and migration planning.
- **Sortable table** — Sort by any column. Configurable page sizes (25, 50, 100).
- **Share filters** — Share your current filter view with colleagues via a copyable text string.
- **Description column** — Toggle to show each field's description directly in the table.

## Field classifications

| Classification | What it means |
| --- | --- |
| **Active** | On screens, has contexts configured, used within the last year. |
| **Stale (>6 months)** | On screens with contexts, but unused for 6+ months. Trending toward disuse. |
| **Stale (>1 year)** | On screens with contexts, unused for over a year. |
| **Likely Unused** | Missing screen associations or context assignments, or Jira does not track its usage. |
| **Unused** | Zero screens and zero contexts. Not visible anywhere in Jira. |
| **System** | Built-in Jira field managed by Atlassian. Cannot be deleted. |
| **Not Tracked** | Team-managed-only field where Jira does not provide usage tracking. |

## Security & privacy

- **Read-only access** — Field Scout only reads field metadata. It cannot modify, create, or delete any fields or data.
- **No data storage** — Nothing is stored on any server. All data is fetched in real-time and displayed in your browser.
- **Admin-only** — Only Jira site administrators can access Field Scout.
- **Runs on Atlassian** — Fully hosted on Atlassian Forge. No external data transfers.

## Getting started

1. Install Field Scout from the Atlassian Marketplace.
2. In Jira, go to **Settings** (gear icon) → **Apps**.
3. Select **Field Scout** from the sidebar.
4. Your custom field analysis loads automatically.
5. (Optional) Click **Load project data** to see per-project field counts and discover team-managed-only fields.
